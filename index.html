ℹ️ Master data — what lives here

Master data is the catalogue this app uses everywhere: Create PO (supplier list, item search, default unit price), Stock On Hand (product identity by code/barcode/name), and transfers / returns. Keep IDs and item codes stable so history and stock stay aligned.

Suppliers
  • Supplier ID — short unique code (e.g. SUP001). Used in imports and item linking; avoid changing it after you have POs.
  • Name — display name on POs and filters.
  • CSV/Excel columns: id / supplier_id, name / supplier_name.
Products / items
  • Item code — primary key in this tool; must be unique.
  • Barcode — optional; speeds search and stock import matching.
  • UOM — pack label (e.g. CRTN, CRTN-24). Receipts use qty × pack size as pieces (PCS) when the name includes a number (e.g. CRTN-24 → 24 PCS per line unit).
  • Unit price — default KWD price on new PO lines (you can still override on the PO).
  • Supplier — ties the item to one vendor for PO filtering and defaults.
  • Import columns: barcode, code / item_code, name, unit / uom, price, supplier / supplier_id.
Save & backups: Use Save Master Data or Save All Data in the header to persist to browser storage. Saving master data also triggers a JSON backup download. Clear actions are destructive — export or back up first.
